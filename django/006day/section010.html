django框架课程

多人开发

技术总监或者经理创建完成仓库后，接下来项目组成员就要进行开发工作了，以下操作由每个组员独自完成

生成SSH公钥

生成公钥的步骤和前面是一样的，将生成的公钥发给经理，经理将公钥添加到github网站对应账号中，组员就可以克隆项目和提交代码了。

本地克隆

克隆的步骤和前面是一样的，默认克隆下来的master分支

创建分支

组员开发需要创建自己的分支，分支的命名各个公司要求不一样，可以用自己的名字命名分支

git branch tom

创建完后推送到远程，组员的远程分支可以对组员的工作做一个备份，如果组员电脑中的代码丢失，就可以通过远程仓库找回。

git push origin tom

同步dev分支到本地

将github上的dev分支同步到本地，因为开发过程中，所有组员都向这个分支上提交阶段性代码，并从这个分支获取最新代码。

git checkout -b dev origin/dev

开发

切换到自己的分支，开始写代码开发

提交代码

组员开发完成后，先提交到自己的分支，推送到远程，接下来需要合并到dev分支，在合并到dev分支前，一定要先将dev同步远程的dev分支，因为在这个组员开发期间，远程的dev分支很可能已经更新了。

切换到dev分支
git checkout dev

拉取远程仓库更新
git pull

合并组员分支到dev分支
git merge tom

将dev推送到远程分支
git push origin dev
